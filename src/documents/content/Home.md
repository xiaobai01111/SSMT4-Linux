# Home

最后更新：2026-03-08

欢迎使用 SSMT4-Linux。

这是一个面向 Linux 桌面环境的第三方游戏启动与管理工具，主要用于管理游戏配置、下载资源、维护 Wine/Proton 运行环境、查看日志，以及在必要时接入 DXVK、VKD3D、Bridge、3DMigoto 等扩展能力。

## SSMT4-Linux 能做什么

- 管理多个游戏配置，并在 Home 页面快速切换当前目标游戏。
- 为不同游戏分别维护主程序路径、运行环境、Prefix 和图形组件配置。
- 提供游戏下载、运行环境管理、日志查看和文档中心。
- 在明确风险的前提下，支持 Bridge、3DMigoto、Mods 和其他实验性能力。

## 适合的使用场景

- 你希望在 Linux 上统一管理多款 Windows 游戏。
- 你需要为不同游戏分别配置 Proton、DXVK、Prefix 或启动参数。
- 你需要一个可视化界面来排查启动失败、路径错误、依赖缺失和日志问题。

## 不适合的使用场景

- 你希望获得官方支持、官方兼容承诺或零风险运行环境。
- 你无法接受 Wine/Proton、图形层、注入层和实验性功能带来的兼容性与账号风险。
- 你打算使用本项目从事违反平台条款、用户协议或反作弊规则的行为。

## 首次使用建议流程

1. 完成初始化向导，确认数据目录与风险确认。
2. 在游戏库中添加或导入目标游戏配置。
3. 在游戏设置中配置主程序路径、运行环境和必要的系统选项。
4. 在设置页面准备好 Proton、DXVK 等运行组件。
5. 回到 Home 页面启动游戏，首次运行后再根据实际情况微调。
6. 若需要使用 Mods、Bridge 或 3DMigoto，请先阅读风险与相关说明文档。

## 主要页面说明

- `Home`：主启动入口。用于快速启动当前选中的游戏，并展示下载或运行状态。
- `游戏库`：集中管理游戏条目、导入模板、删除配置以及切换当前游戏。
- `游戏设置`：针对单个游戏调整主程序路径、运行环境、系统选项和实验性配置。
- `设置`：管理 Proton、DXVK、文档入口、3DMigoto 全局开关等全局功能。
- `文档`：查看随程序打包的本地 Markdown 文档，并可跳转到 GitHub Wiki。
- `Mod 管理`：仅在启用 3DMigoto 全局功能后显示，用于按游戏管理 Mod 与 ShaderFixes。

## 常见工作流

### 1. 已经装好游戏，只想启动

1. 在游戏库中添加游戏。
2. 在游戏设置中填好主程序路径。
3. 选择可用的 Wine/Proton 版本。
4. 检查当前 Prefix 是否安装了 DXVK。
5. 回到 Home 页面启动游戏。

### 2. 需要下载或重新配置游戏

1. 在游戏库中确认游戏条目存在。
2. 打开下载或防护相关入口，完成主程序获取与校验。
3. 回到游戏设置中确认可执行文件路径是否正确。
4. 再次检查运行环境与 Prefix。

### 3. 需要启用实验性功能

1. 先阅读 [《服务条款》与风险声明](00-服务条款与风险声明.md)。
2. 再阅读 [项目风险与要求](01-项目风险与要求.md)。
3. 在 `设置 -> 3DMIGOTO 管理` 中确认风险并启用全局开关。
4. 再阅读 [Mod 管理与 3DMigoto / Bridge 高级工作流](10-Mod-管理与-3DMigoto-Bridge-高级工作流.md)。
5. 只对需要的游戏单独开启，不要默认全局长期打开所有实验性能力。

## 出现问题时先做什么

- 先在 Home 中打开“游戏日志窗口”，再复现问题，避免丢失当前会话的启动日志。
- 优先查看游戏日志窗口、`Player.log`、启动器日志和 Bridge 输出。
- 确认主程序路径是否指向正确的 `.exe`。
- 确认当前游戏选择的 Proton 版本、Prefix 和 DXVK 状态是否匹配。
- 若启用了实验性功能，先尝试关闭相关开关再复测。
- 反馈问题时尽量附带日志、绝对路径、时间点和复现步骤。
