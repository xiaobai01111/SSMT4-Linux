# 环境准备、依赖与底层运行栈

最后更新：2026-03-08

这篇文档解释 SSMT4-Linux 在 Linux 上运行时真正依赖哪些环境、组件、系统包和底层链路。

它不是“装几个包就完事”的清单，而是回答这些问题：

- 项目本身依赖什么；
- 游戏运行依赖什么；
- Proton、Prefix、DXVK、VKD3D、Bridge 分别处在什么层；
- 缺少某个组件时，项目会如何退化、回退或直接报错；
- 普通用户和项目开发者各自应该准备什么环境。

## 1. 先理解整条运行栈

SSMT4-Linux 不是单一程序，而是一条叠起来的运行链。

从下到上可以理解成：

1. Linux 宿主系统
2. 显示服务器与图形驱动
3. Vulkan / GPU 设备与诊断工具
4. Steam / Steam Linux Runtime / umu-run / bwrap 等容器与运行器
5. Wine / Proton 版本本体
6. 每个游戏自己的 Prefix
7. DXVK / VKD3D / winetricks 运行库
8. 游戏本体、启动器与厂商补丁
9. 可选的 3DMigoto / Bridge / Jadeite / Mods
10. SSMT4-Linux 前端与 Tauri 桌面层

出问题时，不要把这十层混成一层看。很多“游戏启动不了”其实根本不是游戏自身问题，而是更下层的 Vulkan、容器或 Prefix 没准备好。

## 2. 项目本身的技术栈

当前项目分三部分。

### 2.1 前端层

前端基于：

- `Vue 3`
- `Vue Router`
- `Vue I18n`
- `Element Plus`
- `Vite`

这部分负责：

- Home、设置、文档、日志、Mod 管理等界面；
- 与 Tauri IPC 交互；
- 本地 Markdown 文档渲染与打包。

### 2.2 桌面层

桌面层基于：

- `Tauri v2`

当前用到的插件包括：

- `tauri-plugin-dialog`
- `tauri-plugin-fs`
- `tauri-plugin-opener`
- `tauri-plugin-shell`
- `tauri-plugin-single-instance`

这层负责：

- 前后端桥接；
- 文件、目录、对话框和外部命令调用；
- 单实例与本地资源协议。

### 2.3 核心能力层

后端基于：

- `Rust`
- `Tokio`
- `Reqwest`
- `Rusqlite`
- `Tracing`

这层负责：

- 游戏扫描与配置；
- 下载、校验、修复；
- Wine / Proton / Prefix / DXVK / VKD3D 管理；
- 启动链与环境变量构建；
- 日志与运行状态收集；
- 3DMigoto / Bridge 配置生成。

## 3. 普通用户真正需要准备的环境

如果你只是要使用项目，不打算开发，也不打算自己编译 Windows Bridge，那么最低要准备的是下面这些层。

### 3.1 宿主桌面环境

至少需要：

- 一个正常可用的 Linux 桌面环境；
- X11、Wayland 或 XWayland 中至少一种能正常工作；
- 能正常显示 Tauri/WebKitGTK 窗口。

当前项目会探测：

- `XDG_SESSION_TYPE`
- `WAYLAND_DISPLAY`
- `DISPLAY`
- `XDG_CURRENT_DESKTOP`
- `DESKTOP_SESSION`

因此：

- 你是什么显示协议，项目是会知道的；
- 如果系统层图形会话本身就异常，项目不会替你修复这层问题。

### 3.2 GPU 驱动与 Vulkan

对绝大多数 Windows 游戏来说，这是最关键的底层依赖之一。

当前项目会调用：

- `vulkaninfo`
- `lspci`

来探测：

- Vulkan 是否可用；
- Vulkan 版本；
- GPU 驱动类型；
- 系统里有哪些显卡。

这意味着：

- 没有 `vulkaninfo` 时，项目对 Vulkan 的判断会退化；
- 没有 `lspci` 时，多显卡识别、驱动分类和 GPU 选择会退化；
- Vulkan 真不可用时，DXVK / VKD3D 这条链通常就没有稳定前提。

### 3.3 Steam 运行时与容器组件

当前项目并不只支持“直接跑 Proton”。

运行器层至少涉及：

- Steam Proton
- `umu-run`
- `pressure-vessel`
- `bwrap`

其中：

- `pressure-vessel` 依赖 Steam 安装的 `SteamLinuxRuntime_sniper`，找不到时会回退到 `SteamLinuxRuntime_soldier`；
- `umu-run` 找不到时，会自动回退到 Proton 启动链；
- `bwrap` 只在你启用沙盒时才是硬依赖，缺失时会直接报错，而不是静默跳过。

所以：

- 想用容器化运行时，Steam 本体和 Steam Linux Runtime 很重要；
- 想用沙盒，必须保证系统里存在 `bwrap`；
- 想用 `umu-run`，必须先装好它本身。

### 3.4 Wine / Proton 本体

当前项目会扫描这些来源：

- 系统 `wine` / `wine64`
- Steam 官方 Proton
- `compatibilitytools.d`
- Lutris Wine
- SSMT4 自己下载的 runners
- 你手动添加的自定义路径

因此：

- 你不一定非要只用 Steam 的 Proton；
- 但你至少要有一套可被扫描到的 Wine / Proton；
- 若配置引用了一个已不存在的版本，启动时会直接报错。

### 3.5 Prefix、DXVK 与 VKD3D

每个游戏都不是直接“裸跑”。

项目会为每个游戏准备：

- `prefix/`
- `prefix/pfx/`
- `prefix.json`

再在这个 Prefix 里安装：

- DXVK
- VKD3D
- winetricks 运行库

如果游戏走 D3D11 路线而 Prefix 里没有 DXVK，经常会出现：

- 黑屏；
- 启动失败；
- 首屏卡死；
- 进入游戏后异常卡顿。

如果游戏需要 D3D12 转译而没有 VKD3D，则 D3D12 链路通常不会正常工作。

### 3.6 运行库与字体

当前内置的运行库列表包括：

- `vcrun2005`
- `vcrun2008`
- `vcrun2010`
- `vcrun2012`
- `vcrun2013`
- `vcrun2015`
- `vcrun2022`
- `dotnet35`
- `dotnet40`
- `dotnet48`
- `d3dx9`
- `d3dx10`
- `d3dcompiler_43`
- `d3dcompiler_47`
- `cjkfonts`
- `fakechinese`
- `mf`
- `xact`

这些运行库通过：

- `winetricks`

安装到 Prefix 里。

也就是说：

- 你要装运行库，系统里必须先有 `winetricks`；
- 没有 `winetricks` 时，项目不会神奇地替你装这些组件。

## 4. 推荐按层准备哪些系统包

这部分不是“唯一正确答案”，而是按当前项目真实实现整理出来的建议层次。

### 4.1 核心桌面依赖

这一层的目标是：让 Tauri 应用本身能稳定启动。

常见核心包包括：

- `gtk3`
- `webkit2gtk-4.1`
- `libsoup3`
- `xdg-utils`

缺这层时，表现通常不是“游戏有问题”，而是：

- 主程序窗口打不开；
- WebKit 相关页面异常；
- 外部打开目录、链接、文件失败。

### 4.2 游戏兼容层基础依赖

这一层的目标是：让 Wine / Proton / Prefix 真正能用起来。

建议至少准备：

- `wine`
- `winetricks`
- `git`
- `unzip`
- `7zip`

其中：

- `winetricks` 用于安装 Prefix 运行库；
- `git` 对 `Data-parameters` 的获取、自愈和更新非常重要；
- 解压工具对下载组件和包处理是基础依赖。

### 4.3 图形与诊断依赖

这一层的目标是：让图形栈可验证、可排查。

建议准备：

- `vulkan-tools`
- `pciutils`

因为项目会使用：

- `vulkaninfo`
- `lspci`

缺这层时，项目不是完全不能运行，但会出现：

- Vulkan 状态无法准确判断；
- GPU 设备枚举不完整；
- 多显卡选择和日志诊断质量下降。

### 4.4 高级运行与容器依赖

这一层按需安装。

建议项包括：

- `steam`
- `umu-launcher`
- `bubblewrap`
- `mangohud`
- `gamescope`
- `gamemode`

它们分别对应：

- Steam Proton 与 Steam Linux Runtime
- `umu-run`
- `bwrap` 沙盒
- MangoHud 性能覆盖
- Gamescope 外层显示容器
- GameMode 性能调度

注意：

- 不是所有用户都必须装这层；
- 但你一旦在项目里启用对应功能，这些就会从“可选”变成“实际依赖”。

### 4.5 输入法与语言环境

项目会检查：

- `XMODIFIERS`
- `GTK_IM_MODULE`
- `QT_IM_MODULE`

并允许你按游戏写入：

- `LANG`
- `LC_ALL`

所以如果你在意：

- 中文输入；
- CJK 字体；
- 游戏语言；

就不能只关注 Proton 和 DXVK，也要准备好系统输入法和字体环境。

## 5. 项目开发者还需要准备什么

如果你要开发这个项目，而不是仅仅使用它，还要再准备一层“构建环境”。

### 5.1 前端构建环境

当前前端要求：

- `Node.js`
- `corepack`
- `pnpm`

仓库也明确要求：

- 使用 `pnpm`

而不是 `npm` / `yarn` 混装。

常用命令包括：

```bash
corepack enable
pnpm install
pnpm run dev
pnpm run build
```

### 5.2 Rust / Tauri 构建环境

当前桌面端要求：

- `Rust stable`
- `Cargo`
- `Tauri CLI v2`

并依赖系统侧的 Tauri 构建库，例如：

- `gtk3`
- `webkit2gtk`
- `libsoup3`

如果你能跑前端、但 Tauri 打不开或打包失败，问题通常就不在 Vue，而在这层原生桌面依赖。

### 5.3 Windows Bridge 构建环境

如果你要自己构建：

- `Windows/ssmt4-bridge.exe`

那么还需要：

- `mingw-w64`
- `cmake`
- `make`

当前 Windows Bridge 是一个独立的 C++ 工程，负责：

- 读取 `bridge-config.json`
- 更新 `d3dx.ini`
- 优化 `Mods` / `ShaderFixes`
- 处理 DLL 部署与注入
- 在同一 Proton 容器里启动并桥接游戏

所以：

- 它不是前端资源；
- 也不是 Rust 自动生成的附属物；
- 缺它时，启用 3DMigoto 的游戏会直接报桥接程序不存在。

## 6. 项目运行时到底会检查什么

理解这一点很重要，因为很多人以为“我没装某个包，项目应该自动帮我兜底”。

当前项目的真实行为更接近：

- 能检查的先检查；
- 能回退的回退；
- 回退不了的直接报错；
- 但不会无限制替你修系统环境。

### 6.1 `umu-run`

若启用了 `umu-run` 但系统找不到它：

- 项目会回退到 Proton 启动链。

这是“可回退依赖”。

### 6.2 `pressure-vessel`

若启用了 `pressure-vessel` 但找不到 Steam Linux Runtime：

- 项目会回退到直连 Proton。

这也是“可回退依赖”。

### 6.3 `bwrap`

若启用了沙盒但系统没有 `bwrap`：

- 项目会直接报错。

这是“不可回退依赖”。

### 6.4 `winetricks`

若系统没有 `winetricks`：

- 运行库安装功能会直接失败。

这不会阻止你打开项目，但会阻止你通过项目安装 Prefix 运行库。

### 6.5 `vulkaninfo`

若系统没有 `vulkaninfo`，或者 Vulkan 本身不可用：

- Vulkan 检测会失败或退化；
- DXVK / VKD3D 的问题会更难定位；
- 某些游戏即使能启动，也未必处于正确图形链。

### 6.6 `lspci`

若系统没有 `lspci`：

- GPU 枚举和驱动判断会退化；
- 多显卡选择提示会不准确；
- 一部分图形问题会更难分析。

## 7. 游戏依赖和项目依赖不是一回事

这是很常见的误区。

### 7.1 项目依赖

指的是让 SSMT4-Linux 自己运行起来所需的东西，例如：

- Tauri 桌面依赖
- WebKitGTK
- 前端构建工具
- Rust / Cargo

### 7.2 游戏依赖

指的是让某个具体游戏在 Prefix 内正常运行的东西，例如：

- DXVK
- VKD3D
- `vcrun*`
- `.NET`
- `d3dcompiler_*`
- `mf`
- `xact`
- `cjkfonts`

### 7.3 底层运行依赖

这是介于两者之间、但又决定成败的一层，例如：

- Vulkan 驱动
- Steam Linux Runtime
- `umu-run`
- `bwrap`
- GPU 选择相关环境变量
- 输入法与字体环境

如果你只装了项目依赖，但没装游戏依赖和底层运行依赖，主界面可能能打开，但游戏照样启动失败。

## 8. 推荐的环境准备顺序

不要一上来就装一堆 Mod、脚本和桥接工具。先把底座搭稳。

### 8.1 普通用户建议顺序

1. 先确认桌面环境、显卡驱动和 Vulkan 正常。
2. 再确认 Steam 或你要使用的 Proton 来源可被项目扫描到。
3. 再创建 Prefix，并安装 DXVK / VKD3D。
4. 按需要补 `winetricks` 运行库。
5. 启动一次游戏，确认基础链路正常。
6. 最后再启用 `umu-run`、沙盒、MangoHud、3DMigoto、Bridge 等高级能力。

### 8.2 开发者建议顺序

1. 先确保 `pnpm install`、`pnpm run build` 能正常通过。
2. 再确保 `cargo check`、Tauri 构建链可用。
3. 若涉及 3DMigoto，再单独确认 `Windows/build.sh` 能构建 `ssmt4-bridge.exe`。
4. 最后再做打包与安装包构建。

## 9. 常见误区

### 9.1 “有 Wine 就够了”

不对。

你还可能缺：

- Proton
- Vulkan
- DXVK
- VKD3D
- winetricks
- Steam Linux Runtime

### 9.2 “主界面能打开，说明环境已经没问题”

不对。

这最多只能说明：

- Tauri / WebKitGTK 层基本可用。

它不能说明：

- Vulkan 正常；
- Proton 可用；
- Prefix 完整；
- 游戏依赖齐全；
- 容器链可用。

### 9.3 “启用了某功能，项目就会自动帮我补好所有系统依赖”

不对。

项目能做的是：

- 扫描；
- 提示；
- 在部分场景回退；
- 在 Prefix 内安装部分运行库；

但不会替你完整管理宿主系统的软件包生态。

## 10. 一张简单的准备清单

如果你想快速自检，可以按这份清单过一遍。

### 普通使用者

- 桌面环境正常
- GPU 驱动正常
- `vulkaninfo` 可用
- `lspci` 可用
- 至少一套 Wine / Proton 可被扫描到
- Steam Linux Runtime 可用（若要用 `pressure-vessel`）
- `winetricks` 已安装（若要装运行库）
- `bwrap` 已安装（若要开沙盒）
- Prefix 已创建
- DXVK / VKD3D 已按需安装

### 项目开发者

- `Node.js`
- `corepack`
- `pnpm`
- `Rust stable`
- `Cargo`
- Tauri 原生依赖
- `mingw-w64 + cmake + make`（若要构建 Bridge）

环境准备不是“装一个 Proton”这么简单；SSMT4-Linux 真正依赖的是“宿主系统图形栈 + 容器运行器 + Wine/Proton + Prefix + 图形转译层 + 运行库 + 可选桥接链”的整套底层组合。
